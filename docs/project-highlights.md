# 项目亮点总结

## 1. 分层设计

项目按职责拆分为配置层、请求层、API 封装层、测试层和数据层：

- 配置层负责读取环境变量和统一管理基础配置
- 请求层负责统一封装 HTTP 请求、请求头、超时和日志输出
- API 层按业务模块封装 `UserApi`、`RepoApi`、`IssueApi`
- 测试层聚焦场景编排与断言
- 数据层通过 YAML 管理测试数据

这样的结构使测试代码与接口逻辑保持解耦，更方便维护和扩展。

## 2. YAML 数据驱动

项目已将用户鉴权、仓库创建、Issue 创建等场景改造成 YAML 数据驱动模式，带来的价值包括：

- 测试逻辑与测试数据分离
- 新增场景时减少重复代码
- 提升测试可维护性

## 3. MySQL 数据库一致性校验

项目当前已实现两条关键数据库校验：

1. 创建仓库后校验 `repository` 表中的记录
2. 关闭 Issue 后校验 `issue` 表中的状态变化

这一部分让测试结果不再停留在接口响应层，而是进一步验证业务动作是否真正生效。

## 4. pytest marker 分组执行

项目支持以下 marker：

- `smoke`
- `db`
- `repo`
- `issue`
- `auth`
- `regression`

通过 marker，可以更接近真实团队的执行方式，例如快速执行核心链路或单独执行数据库校验场景。

## 5. 报告与 CI

项目支持 pytest HTML 报告、Allure 报告以及 GitHub Actions 持续验证：

- 本地执行后可生成可视化报告
- 提交代码后可自动拉起环境并执行测试
- 运行结果可通过 Actions artifact 查看
