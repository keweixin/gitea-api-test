# 数据库校验说明

## 一、文档定位

本文件用于说明当前项目为什么引入数据库校验、已经实现了哪些数据库校验，以及这些校验在整个接口自动化测试项目中的作用。

## 二、数据库环境

当前数据库校验基于 Gitea 使用的 MySQL 环境完成，核心信息如下：

- 接口服务：Gitea
- 数据库类型：MySQL 8
- 数据库名称：`gitea`
- 数据库辅助工具：`common/mysql_helper.py`

## 三、数据库校验的作用

接口自动化测试不仅要验证 HTTP 响应，还需要验证业务动作是否真正写入数据库。当前项目引入数据库校验，主要用于完成以下验证：

1. 验证接口响应结果与数据库结果是否一致
2. 验证关键业务动作是否真正落库
3. 提升测试结果的真实性和说服力

## 四、当前已完成的数据库校验场景

### 1. 创建仓库后校验数据库

对应文件：

- `testcases/test_repo.py`

校验逻辑：

1. 调用接口创建测试仓库
2. 在 `repository` 表中按 `owner_name` 和 `name` 查询目标记录
3. 校验记录存在
4. 校验 `description` 与 `is_private` 与接口请求一致

### 2. 关闭 Issue 后校验数据库

对应文件：

- `testcases/test_issue.py`

校验逻辑：

1. 准备测试 Issue
2. 调用接口关闭 Issue
3. 关联查询 `issue` 表与 `repository` 表定位目标记录
4. 校验 `is_closed` 已更新
5. 校验 `closed_unix` 已写入关闭时间

## 五、执行方式

数据库校验场景已通过 pytest marker 单独标记，可直接执行：

```bash
pytest -m db -v
```

## 六、结论

当前项目已经从“只验证接口返回结果”提升到“接口返回 + 数据库落库联合校验”的层面。这部分能力能够较直接地体现项目的业务验证深度，也是项目中含金量较高的一部分。
