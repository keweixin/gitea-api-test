# 项目亮点总结

## 1. 分层设计

项目已完成基础分层，主要包括：

- 配置层：统一读取 `.env` 中的环境变量
- 请求层：统一封装 HTTP 请求逻辑、请求头与超时时间
- API 层：按业务模块封装 `UserApi`、`RepoApi`、`IssueApi`
- 测试层：使用 pytest 编写测试用例与断言

这一结构使测试代码和接口逻辑保持清晰分离，便于维护和扩展。

## 2. YAML 数据驱动

项目已将用户鉴权、仓库创建、Issue 创建等场景改造成 YAML 数据驱动。

这一方式实现了：

- 测试数据与测试逻辑分离
- 场景扩展时减少重复代码
- 测试维护成本降低

## 3. MySQL 数据库一致性校验

项目当前已完成两条数据库一致性校验：

1. 创建仓库后校验 `repository` 表落库结果
2. 关闭 Issue 后校验 `issue` 表状态变化结果

这一部分体现了接口测试不只停留在响应校验，而是进一步关注业务动作是否真正生效。

## 4. pytest marker 分组执行

项目已支持以下 marker：

- `smoke`
- `db`
- `repo`
- `issue`
- `auth`
- `regression`

当前可按场景执行，例如：

```bash
pytest -m smoke -v
pytest -m db -v
```

这使项目具备了更接近真实测试团队执行方式的组织能力。
